How It All Works — Step by Step

EVE-NG → Ubuntu VM → GitHub → GitHub Pages

1
🖥 You power on the lab in EVE-NG (192.168.1.100)
16 virtual devices boot up inside EVE-NG PRO on your Windows PC.
Each device gets a Telnet console port assigned by EVE-NG (e.g. port 32769, 32770…).
🗂 labs/lab01-ospf-multiarea-acl/ 📄 inventory/hosts.yaml — device IPs & ports
2
⏱ Cron job fires every 5 minutes on Ubuntu VM (192.168.1.103)
A cron entry runs */5 * * * * python3 /...monitor.py
This is the trigger — no manual action needed once set up.
🕐 crontab -e (on Ubuntu VM) 📋 /var/log/monitor.log
3
🐍 monitor.py runs — connects to all 16 devices via Telnet
Uses ThreadPoolExecutor to poll all 16 devices in parallel.
Sends CLI commands: show ip int brief, show ip ospf neighbor, show ip route, ip dhcp
VPCs get show ip via their own console ports.
🐍 labs/lab01-ospf-multiarea-acl/scripts/monitor.py
4
💾 monitor.py writes results → docs/status.json
All parsed output (interfaces, OSPF neighbors, DHCP leases, device status) is saved as JSON.
This file is the single source of truth for the dashboard.
📊 docs/status.json ← updated every 5 min
5
📤 monitor.py does git add → commit → push
At the end of every run, monitor.py automatically:
git add docs/status.json → git commit -m "monitor: ..." → git push origin main
This is your "CI" step — code/data automatically committed and pushed.
🔁 git push → github.com/netauto-abhijeet/CCIE-Automation 🌿 branch: main
6
☁️ GitHub receives the push — GitHub Pages auto-publishes /docs
GitHub is configured to serve the /docs folder as a static website.
Every push = instant publish. No build step, no pipeline config needed.
This is your "CD" step — deployed automatically on every push.
📁 docs/ ← GitHub Pages root ⚙️ Settings → Pages → Source: /docs
7
🌐 Browser opens dashboard — index.html fetches status.json
index.html is a static page — it never changes.
On load it fetches status.json from the same /docs URL and renders all 5 tabs.
⚠ If EVE-NG is OFF → last pushed status.json is shown (stale but accessible).
🌐 docs/index.html ← static UI, never changes 📊 docs/status.json ← fetched at runtime 🔗 netauto-abhijeet.github.io/CCIE-Automation
✅

End result: A live network dashboard, updated automatically every 5 minutes, publicly accessible — zero manual steps after initial setup.

Everything lives in one repo. All work is under labs/lab01-ospf-multiarea-acl/ and docs/. Nothing else is used.

Where "CI/CD" fits in

CI Continuous Integration
Steps 2–5: cron fires → monitor.py runs → commits fresh data to GitHub automatically, on a schedule.
No human manually commits.
CD Continuous Deployment
Steps 5–6: every git push → GitHub Pages instantly re-publishes /docs as a live website.
No manual deploy step.
Full flow in one line
EVE-NG → monitor.py → status.json → git push → GitHub Pages → Dashboard
